1. Для инициализации репозитория пишем: git init . (точка - текущая папка, можно указывать полный путь).

2. Чтобы понять текущее состояние репозитория пишем: git status.

3. Чтобы добавить файлы в staging area пишем: git add "имя файла или . (добавятся все измененные файлы из текущей папки)".

4. Чтобы сделать commit пишем: git commit -m "Описание коммита" (можно закоммитить и без "-m").

5. Чтобы посмотреть историю коммитов пишем: git log.

6. Чтобы перейти к конкретному коммиту пишем: git checkout "hash коммита, к которому переходим".

7. Чтобы связать удаленны репозиторий с локальным пишем один раз: git remote add origin "адрес удаленного репозитория".

8. Чтобы просмотреть список удаленных репозиториев пишем: git remote --v.

9. Чтобы залить код на удаленный репозиторий пишем: git push origin master (ветку, которую заливаем).

10. Чтобы поддтянуть изменения с удаленного репозитория: git pull origin master (ветку, в которую подтягиваем с удаленного репозитория).

11. Чтобы посмотреть в какой ветке мы находимся в данный момент: git branch.

12. Чтобы создать новую ветку и переключится на нее: git checkout -b "name branch".

13. Чтобы переключится между ветками: git checkout "name branch".

14. Чтобы слить две ветки, нужно перейти на ту ветку, в которую производится слияние и написать команду: git merge "название той ветки, которую нужно слить с текущей веткой".

15. Удаление ветки: git branch -d "name branch".
